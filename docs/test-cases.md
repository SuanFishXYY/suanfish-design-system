# 算鱼设计系统 v4.2.6 · 测试用例集

> 5 个真实任务 · 完整走 **六步圣人议会协议** · 给出**预期 4:4:4 议会评分 + 召唤清单 + 投票结果 + 三段式输出样例**, 供后续手动或自动化验证可重复对照。
>
> **诚实声明**: 打分/投票数字由作者预设, 用于教学与回归对照, 非真 multi-agent runtime 输出。完整旗舰演示见 [`v4.2-congress-simulation.md`](v4.2-congress-simulation.md)。

## 🧭 使用说明

每个测试用例包含:

- **任务输入** — 用户原话 + `task_kind` (visual / motion / structural / philosophical / mixed)
- **Step 1-3 · 4:4:4 议会评分预期** — 12 位 Tier 0 (4 哲 + 4 艺 + 4 音) 五维加权, 标出 ≥ 7.5 入场者 (三大类强制各 ≥ 1)
- **Step 4 · 跨类邀请预期** — 入场圣人从 420 板凳跨三大类邀请的助手 (#NNN 哲 / #ANNN 艺 / #MNNN 音 · cap 15)
- **Step 6 · 投票预期** — Tier 0 = 2 票 + 类别匹配 +0.5 / 助手 = 1 票 · ≥ 2/3 通过 · 禁一票否决
- **三段式样例** — 理论依据 / 设计方向 / 改造动作
- **预期 Path 路径** — A-G 哪条 · **预期主导 agent** — 下游接手者

如 skill 实际运行与本文档预期偏差较大 (> 30%) → 视为 regression, 需调查提 issue。

## 🧪 如何使用本测试集

### 手动验证
1. 在 Claude Code / Copilot CLI 装好 plugin (`/plugin install suanfish-design-system`)
2. 复制 TC-XX 的 "任务输入" 原话发给 skill
3. 对比实际输出与本文档"预期" (入场圣人 / 三大类均衡 / Path / 主导 agent)
4. 偏差 > 30% → 视为 regression, 提 issue

### 自动化验证 (未来计划)
- 写 `scripts/verify-test-cases.js`
- 调用 skill 跑 TC-01 至 TC-05
- 提取实际入场圣人清单 vs 预期, 计算召回率 + 校验三大类各 ≥ 1
- 召回率 < 60% 或三大类某类缺席 (4:4:4 骨架被破坏) → 报警
