# 🎬 BRIEF — 新版本登录亮点介绍（v2.2 onboarding）

**作者**：moment-strategist
**派单时间**：v2.2.0 上线日

---

## 一句话框定

让首次进入 v2.2 的用户在 ≤ 15s 内理解 3 个升级点（多模态 / 联网检索 / Gemini 切换），并能随时跳过。

## 模式

`onboarding · 路径 A`

## 六维体检

| 维度 | 取值 | 备注 |
| --- | --- | --- |
| 重复度 | 1-2 次 | 首次登录 + 版本升级 |
| 信息密度 | 序列 · 3 步 | 不能多于 3，否则触发 P-OD1 |
| 交互意图 | 读 + 探索 | 不强制操作 |
| 时长预算 | 10-15s | 单步 3-5s |
| 情感目标 | 期待 + 欢迎 | 不煽情，不焦虑 |
| 复用度 | 100% | 标准 onboarding 模板 |

`onboarding-director`

## 成功标准

- [ ] 3 步骨架，**不超过 3** （P-OD1）
- [ ] 用户可在**任何步骤**点击「跳过」（P-OD2 必查）
- [ ] 整体时长 ≤ 15s（避免 R1）
- [ ] 文案符合"主语-动作-后果"三问（P-CW1）
- [ ] 不出现禁忌词「亲、您好哦、稍后再试」（P-CW2）
- [ ] 仪式动画用 `ease-out` 600-1000ms（仪式时刻允许）
- [ ] 仪式 keyframes **不渗透**到稳态界面（三条铁律 1）

## 为何选这条路径

`[孙子 · 上兵伐谋]` — 自问：如果不做这个 onboarding 会怎样？

答：用户进入 v2.2 后将**自己摸索**新功能。读图 / 联网检索 / Gemini 切换是有发现成本的功能，不主动介绍 → 30% 用户 7 天内不会发现这些升级 → 商业价值损失。

→ **结论：值得做。但克制做。**

派单成立。
