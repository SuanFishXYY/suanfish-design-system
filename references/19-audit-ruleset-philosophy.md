> **v2.3 新增** — 让哲学不只是字符，而是可校验的规则。

> `ui-auditor` 在每次评审时加载本规则集，对 agent 输出做**哲学层校验**。任何 agent 的 SPEC/REPORT 必须能通过这一层。

## 一、通用哲学检查（全员）

### P-G1 · 哲学锚点引用检查

**规则**：任何 SPEC/REPORT 输出必须至少引用 1 个哲学命题或 Canon 编号。
**判罚**：🟧 警告
**示例**：
- ✅ 通过：`[Canon-D5] 该方案删除冗余装饰 → 隐而不见`
- ❌ 不通过：`方案：删除装饰，更简洁` （无引用）

### P-G2 · 引用合理性检查

**规则**：引用的哲学命题必须与决策**逻辑相关**，不能炫学。
**判罚**：🟧 警告
**反例**：在弹窗讨论里引用「赫拉克利特 · 人不能两次踏入同一河」→ 牵强

### P-G3 · 思维链完整性

**规则**：SPEC 中思维链必须包含至少 3 个步骤：
1. 识别用户处境 / 问题
2. 引用哲学锚点 / Canon
3. 推出结论
**判罚**：🟨 提示

### P-G4 · REJECT 必带哲学命题

**规则**：任何 REJECT 输出必须显式标注违反的哲学命题 + 出处。
**判罚**：🟥 严重 → 直接打回
**模板**：
```
🛑 REJECT R{N}
违反命题：[哲学家 · 命题简称]
出处：原典文献
```

## 二、Agent 专项检查

### `moment-strategist`

#### P-MS1 · BRIEF 前置质询
**规则**：派单 BRIEF 前必须问"如果不做这个需求会发生什么？"，答案是"什么都不会发生" → 必须 REJECT。
**锚点**：`[孙子 · 上兵伐谋]`
**判罚**：🟥 严重

#### P-MS2 · R1-R6 哲学命题覆盖
**规则**：触发 R1-R6 时必须列出对应哲学命题（详见 `agents/moment-strategist.md` R1-R6 表）。
**判罚**：🟥 严重

### `onboarding-director`

#### P-OD1 · 步数检查
**规则**：onboarding 步数 > 3 时，必须证明"为什么不能合并"。
**锚点**：`[庄子 · 庖丁解牛]` — 顺势而为，找肯綮
**判罚**：🟧 警告

#### P-OD2 · 跳过权

## 三、东方 / 西方 flavor 切换检查

### P-F1 · Flavor 一致性

**规则**：SKILL.md 的 `flavor:` 字段与实际输出风格必须一致。
- `flavor: eastern` → 输出应低密度 / 隐喻 / 留白
- `flavor: western` → 输出应高密度 / 直白 / 强 CTA
- `flavor: hybrid` → 信息层 Western / 仪式层 Eastern

**判罚**：🟧 警告

### P-F2 · Flavor 禁忌交叉

| Flavor | 禁忌 |
| --- | --- |
| eastern | 倒计时紧迫感、营销弹窗、过度对比色 |
| western | 模糊隐喻、过度留白、纯文学性文案 |

**判罚**：🟥 严重

## 三·补 · v2.5 新增 9 agent · Path G AI-native 专项检查

> *9 位新 agent，对应 27 条新 P-XX 规则。审计时按 agent 触发。*

### `stream-craftsman` (赫拉克利特 · 万物流变)

#### P-SC1 · 首 token 上屏 ≤ 400ms
**规则**：流式接口从用户发送到首字符可见，必须 ≤ 400ms；否则必须显示骨架/cursor 占位。
**判罚**：🟥 严重（赫拉克利特拒绝）—— 让用户盯空白 = 否认"流"的存在

#### P-SC2 · per-token 渲染禁令
**规则**：增量 markdown 必须按"块"提交（段落/列表项/代码块完整后才入 DOM），禁止 per-token 重排。
**判罚**：🟥 严重

#### P-SC3 · Stop 响应 ≤ 100ms
**规则**：用户点击 Stop 后，必须在 100ms 内停止生成，且保留已生成内容不闪烁。
**判罚**：🟧 警告

### `tool-call-presenter` (奥斯汀 · 言语行为)

#### P-TCP1 · 四态可见
**规则**：工具调用必须明示 pending/running/success/error 四态，不可隐藏。
**判罚**：🟥 严重（违反可见性原则）

#### P-TCP2 · 副作用工具二阶段确认
**规则**：write/send/execute/deploy/delete/modify 类工具默认必须先显示"即将调用"+ 完整参数预览，用户确认才执行。
**判罚**：🟥 严重（违反可撤回原则）

#### P-TCP3 · 错误归因到具体环节
**规则**：失败时不允许笼统"出错了"，必须区分 input-error / network / tool-internal / permission。
**判罚**：🟧 警告

### `agent-thread-architect` (博尔赫斯 · 小径分岔)

#### P-ATA1 · regenerate 保留旧版本
**规则**：重新生成后旧版本必须可访问（`◀ 1/N ▶` 切换），禁止直接覆盖。
**判罚**：🟥 严重

#### P-ATA2 · 编辑用户消息走新分支
**规则**：edit-resubmit 必须创建新分支，旧分支保留可回溯。
**判罚**：🟥 严重

#### P-ATA3 · 长线程 checkpoint
**规则**：对话 > 20 轮必须每 5 轮自动 checkpoint，提供回溯入口。
**判罚**：🟧 警告

### `reasoning-visualizer` (笛卡尔 · cogito)

#### P-RV1 · reasoning 默认折叠
**规则**：thinking/reasoning 字段默认折叠为单行摘要（`💭 已思考 N 秒`），不允许默认展开占据 ≥ 30% 视觉。
**判罚**：🟧 警告

#### P-RV2 · reasoning 与答案视觉权重差异
**规则**：reasoning 文字色 / 字号必须**弱于**答案；不允许同等视觉权重。
**判罚**：🟧 警告

#### P-RV3 · 隐私不可选中
**规则**：reasoning 区块默认 `user-select: none`；提供"复制思考"独立入口。
**判罚**：🟨 轻微

### `citation-keeper` (福柯 · 作者功能)

#### P-CK1 · 禁止裸 URL
**规则**：答案中如有引用，禁止裸 URL 平铺；必须用角标/段落级/嵌入式三种形态之一。
**判罚**：🟥 严重

#### P-CK2 · 断链不删除
**规则**：源链接 404/撤回时必须保留引用条目并标 `⚠ 链接失效`，禁止静默删除。
**判罚**：🟥 严重（隐藏失效 = 篡改历史）

#### P-CK3 · hover 预览延迟 ≥ 200ms
**规则**：角标 hover 触发预览延迟 ≥ 200ms，避免移过即弹。
**判罚**：🟨 轻微

### `artifact-architect` (海德格尔 · 作品)

#### P-AA1 · 长产物自动脱出
**规则**：≥ 30 行代码 / ≥ 1500 字 markdown / 任何 SVG 或可预览 HTML，必须脱出对话流进入画布视图。
**判罚**：🟧 警告

#### P-AA2 · 画布可编辑
**规则**：画布产物默认必须可编辑（代码 → CodeMirror / md → 双视图），禁止只读模式作为唯一形态。
**判罚**：🟥 严重（剥夺用户主权）

#### P-AA3 · 关闭前数据保护
**规则**：用户在画布中编辑了内容但未应用 → 关闭画布时必须警告"未保存的修改"。
**判罚**：🟥 严重

### `prompt-input-craftsman` (奥斯汀 · 施为)

#### P-PI1 · IME 安全
**规则**：输入框 Enter 行为必须检测 IME composition 状态，选词中的 Enter 不发送。
**判罚**：🟥 严重（中国用户主权红线）

#### P-PI2 · 草稿持久化
**规则**：未发送内容必须存 localStorage，下次进入显示"恢复 / 丢弃"。
**判罚**：🟧 警告

#### P-PI3 · 附件上传进度
**规则**：上传中的附件卡片必须显示进度条 + 取消按钮，禁止无反馈上传。
**判罚**：🟧 警告

### `rate-limit-communicator` (罗尔斯 · 正义)

#### P-RLC1 · 禁止技术腔
**规则**：用户可见的限制提示禁止出现 "rate limit exceeded" / "quota" / "429" / "token budget" 等术语，必须人性化。
**判罚**：🟧 警告

#### P-RLC2 · 模型降级必告知
**规则**：服务端将请求 fallback 到其他模型时，UI 必须明示"已切换到 X"，不允许神不知鬼不觉。
**判罚**：🟥 严重（信任崩塌）

#### P-RLC3 · 付费墙无 dark pattern
**规则**：付费弹窗禁止默认勾选订阅、禁止次级按钮（"先免费"）被刻意弱化到不可见。
**判罚**：🟥 严重（法律也不允许）

#### P-RLC4 · 历史归档非删除
**规则**：达到上限的历史会话必须明示"归档"（保留只读）而非"删除"；提前 3 天预警。
**判罚**：🟥 严重

### `model-switcher-stylist` (列维-斯特劳斯 · 修补匠)

#### P-MS1 · 切换需告知
**规则**：用户主动或被动切换到不同厂商/能力差异大的模型时，必须有 banner 或 toast 告知。
**判罚**：🟥 严重

#### P-MS2 · 跨模型视觉一致
**规则**：不同模型的回答气泡不可用不同主色；仅左上角 12px 厂商 logo 区分。
**判罚**：🟧 警告

#### P-MS3 · 模型卡必备字段
**规则**：模型选择器中每张卡必须包含：名称 + 厂商 + 速度 + 价格 + 智力 + 上下文 + 能力 + 一句话适合。
**判罚**：🟧 警告

### Path G 横切总规

#### P-G-OVERALL · AI-native 四原则
所有 Path G 相关页面必须同时满足：
1. **认知可视化** ：reasoning / streaming 不藏
2. **输出归因化** ：citation / tool-call 不藏
3. **限制透明化** ：rate-limit / model-switch 不藏
4. **操作可撤回** ：thread / artifact / prompt-input 都可回退

任一不满足 → 🟥 严重（赫拉克利特/奥斯汀/罗尔斯/博尔赫斯 四方拒绝）
