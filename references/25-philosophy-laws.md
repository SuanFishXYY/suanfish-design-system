# 🌀 算鱼设计系统 · 发展规律（Development Laws）

> *"理解事物发展的规律，才能做好系统。"* — v3.0 立项语

本文档是 v3.0 引入的「**Layer 3 · 发展规律**」哲学层。回答的问题：**为什么这个时代的设计长成这样？下一个时代会变成什么？**

---

## 一、为什么需要发展规律？

价值告诉「该选哪边」，辩证告诉「为什么有两边」。发展规律告诉「**矛盾随时间如何演化**」。

```
辩证层：「自动化 ⟷ 掌控 是永恒矛盾」
发展层：「但控制权 50 年来在持续从开发者 → 用户 → AI 转移
         今天选 100% 用户掌控 = 与时代洪流逆流」
```

历史不重复，但押韵。**5 条发展律 = 把"押韵"显式化**。

---

## 二、5 条发展规律

### **L1 · 复杂度螺旋律**

> 系统按 **简单 → 复杂 → 重新简化（但本质不同）** 三段循环演进。

| 阶段 | 表现 | 例子 |
| --- | --- | --- |
| Simple | 直接、能用 | DOS 命令行 |
| Complex | 功能堆砌 | Windows 95 控制面板 800 项 |
| Simplified-as-different | 表面简化但内核更复杂 | iPhone 一个 Home 键 + AI 黑箱 |

**违反信号 R13**：
- 在 Complex 阶段坚持继续加功能，不肯做收口
- 在 Simplified 阶段强行回到 Complex 风格（"专业版"陷阱）

**应用**：dialectical_brief 中标注当前 BRIEF 处在哪个阶段，预警「过早收口」或「拒不收口」。

---

### **L2 · 抽象具象交替律**

> 信息组织在 **file → folder → tag → vector → agent** 这条主线上单向移动，每代抽象层级提高。

| 代 | 组织形态 | 时代 |
| --- | --- | --- |
| 1 | 文件 (1980s) | 一切是文件 |
| 2 | 文件夹 (1995) | 树状层级 |
| 3 | 标签 (2005) | 多对多关系 |
| 4 | 向量 (2018) | 语义相近性 |
| 5 | Agent (2024+) | 信息找人，不是人找信息 |

**违反信号 R14**：
- 2024 年还在做"新建文件夹"为核心交互（Notion 早期被骂的点）
- 反向退化（从 tag 退回 folder）必须有强业务理由

**应用**：B 路径（稳态）BRIEF 中 IA 设计提案如果还在 folder 时代，强提醒。

---

### **L3 · 控制权下移律**

> 系统决策权 50 年来在 **开发者 → 用户 → AI** 这条单向链上转移，每次转移幅度加大。

| 时代 | 谁决定 | 例子 |
| --- | --- | --- |
| 1980s | 开发者 | 编辑器只能 vi 一种模式 |
| 1990s | 高级用户 | 自定义快捷键 / 插件 |
| 2010s | 普通用户 | 个性化推荐设置 |
| 2024+ | AI | Agent 替用户做选择，用户回看可改 |

**违反信号 R15**：
- AI-native 路径（G）BRIEF 中坚持 6 层确认流，不让 AI 自主决策
- 反向：让 AI 决定不可逆操作（删除/支付）— 这是逆方向越权

**应用**：G 路径 BRIEF 默认通过；A-F 路径如要赋予 AI 决策权，必须明确"可撤销 / 可回看"。

---

### **L4 · 反馈循环缩短律**

> 反馈周期 50 年来在 **batch → daily → real-time → predictive → preemptive** 单向缩短。

| 时代 | 周期 | 例子 |
| --- | --- | --- |
| 1970s | Batch | 打孔卡隔夜出结果 |
| 1990s | Daily | 每日构建 |
| 2010s | Real-time | 实时协作（Google Docs） |
| 2020s | Predictive | 输入到一半 AI 已预测下文 |
| 2025+ | Preemptive | AI 主动提示「你接下来可能要…」 |

**违反信号 R16**：
- 仍设计 "请等待，5 分钟后查看结果" 的非异步流（除非物理限制）
- 反向：在 batch 业务（财务月结）强行 real-time 化（无意义压力）

**应用**：流式输出（Path G）BRIEF 默认通过；任何 latency >2s 无 streaming/skeleton/progress 的设计触发 R16。

---

### **L5 · 模态融合律**

> 表达方式按 **text → multimedia → multimodal → synesthetic** 单向融合，每代调用更多感官。

| 代 | 形态 | 例子 |
| --- | --- | --- |
| 1 | 纯文本 (1980s) | BBS |
| 2 | 多媒体 (1995) | 图文 + 视频 |
| 3 | 多模态 (2023) | 文 ↔ 图 ↔ 语音 ↔ 视频 互相转换 |
| 4 | 通感 (2030+) | 触觉 + 嗅觉 + AR 空间感知 |

**违反信号 R17**：
- 2024 年还做"纯文字按钮 + 文字反馈"的核心流程（移动端尤其忌）
- 反向：信息密度场景（dashboard 数字阅读）强行加视频/语音（注意力浪费）

**应用**：G 路径多模态融合 BRIEF 默认对齐；A-F 路径如缺少图标/动效/反馈音，且非密度优先场景，触发 R17。

---

## 三、5 条律之间的关系

```
L1 复杂度螺旋
   ↓ 驱动
L2 抽象交替 (复杂到瓶颈 → 升级抽象层)
   ↓ 触发
L3 控制权下移 (新抽象层人不够 → 让 AI 接管)
   ↓ 加速
L4 反馈循环缩短 (AI 接管后反馈瞬时化)
   ↓ 释放
L5 模态融合 (瞬时反馈 + 多模态成为新基线)
```

5 条律不是独立的，是**驱动链**。L1 是根因，L5 是表象。

| 发展律 | 影响的矛盾（来自 24-dialectics） |
| --- | --- |
| L1 复杂度 | D1 简洁 ⟷ 可发现 |
| L2 抽象 | D3 个性化 ⟷ 一致性 |
| L3 控制权 | D2 自动化 ⟷ 掌控 |
| L4 反馈 | D6 即时 ⟷ 深思 |

发展律是**矛盾的时间维度**——告诉你这对矛盾在哪个时刻正在如何漂移。

## 五、违反规律 → REJECT R13-R17

```
R13 违反 L1 (复杂度阶段错配)
R14 违反 L2 (停留在过时的组织形态)
R15 违反 L3 (不让 AI 该接管的接管 / 让 AI 越权)
R16 违反 L4 (该实时的不实时 / 不该实时的强实时)
R17 违反 L5 (该多模态的单模态 / 该单一的强融合)
```

这 5 条规则是「**逆时代洪流**」预警。违反 ≠ 立即返工，但必须在 dialectical_brief 中注明"已知违反 R1X，业务理由是…"。

---

## 六、规律的局限与边界

> 规律不是命令。规律是趋势。

| 例外 | 不适用律 | 原因 |
| --- | --- | --- |
| 医疗 / 金融关键操作 | L3 (控制权下移) | 不能让 AI 决定剂量 / 转账 |
| 法律合规文档 | L4 (反馈缩短) | 必须留思考时间 |
| 极简书写工具 (iA Writer) | L5 (模态融合) | 单模态是核心价值 |
| 内部工具 | L1 (复杂度螺旋) | 用户是专家，复杂可接受 |

**规律的判断**：当 BRIEF 命中例外，dialectician 必须显式说出"豁免 LX，理由是…"，不能默默违反。

## 七、规律 ⟷ 历史定位

5 条律的存在感来自对**8 大时代**的归纳：

> 详见 [26-historical-positioning.md](26-historical-positioning.md)
> Pre-Web / Web1.0 (拟物) / Web2.0+Mobile First / Flat+Material /
> Glass+Neumorphic / LLM Pre-Native / AI-native / Spatial-Multimodal (未来)

historian agent 拿 BRIEF 找历史坐标，futurist agent 拿规律预测下一站。两个 agent 协同完成"过去 - 现在 - 未来"的辩证三段论。

---

## 八、未来候选规律（v3.x 评估中）

- **L6 · 注意力稀缺律**：用户注意力跨代缩短，争夺成本指数增长
- **L7 · 信任稀缺律**：技术承诺通胀 → 用户对 UI 信任阈值跨代提高
- **L8 · 边界消融律**：私人 ⟷ 公共 / 工作 ⟷ 生活 / 现实 ⟷ 虚拟边界单向溶解

新增律须满足：① 跨 3+ 时代有数据支撑 ② 与现有 5 律不可还原 ③ 至少 1 个 BRIEF 因它触发 REJECT。
