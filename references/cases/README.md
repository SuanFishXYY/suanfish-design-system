# 📚 案例库 · Cases

> *每条规则配 3 正例 + 3 反例。规则会忘，案例会记得。*

本目录按 agent 组织，每个文件包含该 agent 对应的经典设计案例和反例。

## 目录结构

```
cases/
├── README.md（本文件）
├── modal-craftsman.md          · 弹窗决策案例
├── animation-choreographer.md  · 动画克制案例
├── empty-state.md              · 空状态文学案例
├── copy-writer.md              · 文案对比案例
└── data-viz.md                 · 图表减法案例
```

## 使用方法

1. agent 在评审时，可引用 `[Case-XX]` 编号
2. 案例必须包含：**场景 / 决策 / 哲学依据 / Canon 编号**
3. 反例须给出**修正方向**，不止"批评"

## 案例评级

| 标记 | 含义 |
| --- | --- |
| ✅ | 经典正例（10 年后仍站得住） |
| ⚠️ | 有争议（特定场景成立） |
| ❌ | 反例（应修正） |
