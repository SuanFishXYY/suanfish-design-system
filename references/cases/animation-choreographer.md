# 🎬 案例 · animation-choreographer

哲学锚点：**老子 ·「大象无形，大音希声」**

---

## ✅ Case-A1 · iOS 拉动刷新的橡皮筋

**场景**：列表顶部下拉刷新

**决策**：橡皮筋阻尼曲线，松手回弹，转一个小小的菊花。整个动画 < 800ms。

**哲学依据**：
- `[老子 · 大象无形]`：动画存在感极低，用户感受到的是"自然"
- `[Canon-D5]`：unobtrusive
- `[物哀 · 余韵]`：松手那一刻的微弹是余韵，不喧宾夺主

---

## ✅ Case-A2 · macOS Dock 图标放大

**场景**：鼠标划过 Dock 时图标放大

**决策**：缩放是连续的，跟随鼠标位置实时变化。无延迟、无关键帧跳跃。

**哲学依据**：
- 物理真实感 → 让用户感觉是"自然反应"，不是被告知"我做了动画"
- `[Canon-D8]`：thorough down to the last detail

---

## ✅ Case-A3 · 微信支付成功的对勾

**场景**：支付成功瞬间

**决策**：一个 ~300ms 的对勾绘制动画 + 轻量爆破粒子。结束即停，不循环。

**哲学依据**：
- `[物哀 · 短暂]`：仪式感存在于"恰好结束"
- `[Canon-E2]`：仪式应当短暂、克制、留余韵
- 时机重要：和支付成功的语义瞬间精确同步

---

## ❌ Case-A4 · 每个按钮 Hover 都有 600ms 缩放

**场景**：表格里有 50 个操作按钮，每个 hover 都放大动画

**违反**：
- `[Canon-D5]`：用户鼠标滑过表格 → 50 次动画 → 严重打扰
- 高频路径动画 > 200ms = 卡顿感

**修正方向**：
- 高频路径动画 ≤ 200ms
- Hover 改为轻量颜色过渡（背景色 100ms 淡入）

---

## ❌ Case-A5 · 列表加载用粒子飞入

**场景**：每条数据从下方"啵啵啵"飞入位置，时长 600ms 错峰

**违反**：
- `[大象无形]`：用户视线焦点被强行牵引
- 数据展示是稳态高频场景，不是仪式场景
- 显著降低实际感知速度

**修正方向**：
- 改为整体淡入 + 骨架屏过渡（150ms）
- 或直接出现，无动画

---

## ❌ Case-A6 · 全屏过渡用 1.2 秒的炫酷转场

**场景**：每次切换页面，旋转 + 缩放 + 视差，1200ms

**违反**：
- 用户每次切换都要等 1.2 秒
- `[Canon-D9]`：浪费用户时间 = 不环境友好
- `[Canon-D7]`：5 年后这种视效会成为"过时炫技"

**修正方向**：
- 同级页面切换 → 150ms 淡入
- 进入详情页 → 200-300ms 滑入
- 关键仪式时刻才允许长动画

---

## 判定矩阵 · 动画时长上限

| 场景 | 时长上限 | 推荐曲线 |
| --- | --- | --- |
| 微反馈（hover / focus / click） | 150ms | ease-out |
| 元素出现 | 200ms | ease-out |
| 同级页面切换 | 200-300ms | ease-in-out |
| 模态出现 / 抽屉滑入 | 300ms | ease-out |
| 仪式时刻（首次成功 / 里程碑） | 600-1000ms | 自定义 |
| 加载等待 | 持续循环 | linear |

> **铁律**：用户能描述出"刚才有个动画"——动画失败。除非这是仪式时刻。
