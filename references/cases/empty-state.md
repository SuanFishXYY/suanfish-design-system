# 🌌 案例 · empty-state-storyteller

哲学锚点：**海德格尔 ·「Dasein · 此在」**

> 空状态是「存在」的提示，不是「缺失」的报错

---

## ✅ Case-E1 · Things 3 · 「Anything you'd like to do?」

**场景**：今日任务列表为空

**决策**：
- 文案："Anything you'd like to do?"（你想做点什么吗？）
- 视觉：极简插画 + 一行问句
- 动作：下方提示「点 + 创建任务」

**哲学依据**：
- `[Dasein · 此在]`：呈现"此刻可以做什么"
- `[Canon-E1 · 间]`：留白让问句自己说话
- 文案是邀请，不是空虚

---

## ✅ Case-E2 · Notion 新页面 · 命令面板自动浮起

**场景**：新建空白页面

**决策**：光标自动定位，输入 `/` 立即弹出命令面板，给出 20+ 可能动作。

**哲学依据**：
- 不让用户面对纯白恐惧
- 直接给出"可能性的清单"
- `[Canon-D4]`：让产品易于理解

---

## ✅ Case-E3 · GitHub 仓库无 issue · 「Welcome to the issue tracker」

**场景**：仓库刚创建，issue 列表为空

**决策**：
- 标题："Welcome to the issue tracker"
- 副文："Issues are used to track todos, bugs, and ideas."
- CTA：「New issue」按钮 + 「Learn more about issues」链接

**哲学依据**：
- 解释这个空间是什么 → 用户认知建立
- 给出第一步动作 → 此在的可能性
- 提供学习入口 → 余地

---

## ❌ Case-E4 · 「暂无数据」

**场景**：搜索结果为空 / 列表无数据

**违反**：
- 把"空"当作"故障"
- 不给任何下一步指引
- 完全违反"此在"——只呈现缺失，不呈现存在

**修正方向**：
- 搜索为空 → "找不到匹配 'xxx' 的结果。试试更短的关键词，或浏览全部？"
- 列表为空 → "还没有 X。创建第一个？" + CTA

---

## ❌ Case-E5 · 「请稍后再试」

**场景**：网络错误导致列表加载失败

**违反**：
- 把责任推给用户（"稍后再试"=你的问题）
- 不解释发生了什么
- `[Canon-D6]`：不诚实

**修正方向**：
- "暂时连不上服务器。我们会自动重试，或你也可以手动 [刷新]。"
- 给出客观事实 + 自动恢复机制 + 主动权交还

---

## ❌ Case-E6 · 「您还没有任何收藏」+ 卡通哭脸

**场景**：收藏列表为空

**违反**：
- 卡通哭脸 = 情感绑架
- 把"还没收藏"渲染为负面情绪
- `[Canon-D6]`：不诚实——空不是悲伤

**修正方向**：
- 中性平静："收藏你喜欢的内容，方便以后找到。"
- 或邀请性："去发现页看看？" + CTA

---

## 空状态文案模板（Dasein 模式）

```
[识别用户处境]
  └→ 你现在在 X 场景

[呈现此在的可能性]
  └→ 这里可以 Y / Z

[给出第一步]
  └→ [CTA 按钮] / [次要链接]

[可选：留出探索余地]
  └→ "或查看 [其他入口]"
```

> **铁律**：空状态文案禁止出现的词：暂无、还没有、请、稍后、抱歉、Oops。
